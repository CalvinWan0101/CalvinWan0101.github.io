APCS(大學程式設計先修檢定)

在APCS的觀念題中有一種很麻煩的題目是 code tracing

一開始想說應該還好，畢竟也不是沒有看過別人的程式碼，應該跟著跑一次就好了

天真如我

一時找不到之前繞到頭暈掉的題目，隨便舉一個考古題的例子，還算簡單的

若以F(15)呼叫右側F()函式，總共會印出幾行數字？

void F(int n)
{
    printf("%d\n",n);
    if(n%2==1)&&(n>1)
     {
         return F(5*n+1)
     }
    else
    {
        if(n%2==0)
         return F(n/2);
    }
}

解題思路：

當函式輸進一個變數n,此時分為兩種狀況

條件 動作
n為偶數 回傳F(n/2)
n不為偶數 回傳F(5*n+1)

這時候n的變化如下

印出次數 此時的n值
01 15
02 76
03 38
04 19
05 96
06 48
07 24
08 12
09 06
10 03
11 16
12 08
13 04
14 02
15 01

Answer == 15

全題考到的觀念只有

  1. 條件判斷
  2. 簡單的運算
  3. 函數

明明只是考三個很簡單的觀念卻硬要強行繞15次再問考生繞了幾次

讓人不禁懷疑到底出這樣的題目對於程式能力的檢定是否真的有幫助?

我記得我在APCS的社團有看過別人說：想要程式寫得好，請認真看APCS觀念題並引以為戒

也有人說 APCS = A lot of Problems Caused by Spaghetti code

至少在我少少的解題經驗中完全沒有用過這麼沒有邏輯的東西

或者是這題

int g1 = 30;
int g2 = 20;

int f1(int v)
{
	int g1 = 10;
	return g1 + v;
}
int f2(int v)
{
	int c = g2;
	v = v + c + g1;
	g1 = 10;
	c = 40;
	return v;
}
int main()
{
	g2 = 0;
	g2 = f1(g2);
	printf("%d", f2(f2(g2)));
}

很明顯這題考觀念有：

  1. 函數
  2. 變數形態(全域以及區域變數)

除了打算折磨考生的出題者，基本上沒有人寫程式會故意讓全域變數跟區域變數同名

然後今年7月的APCS名額變少，報名的人好像有變多
